# Analog Electronics Open Day Demo — Architecture Document

## Project Overview

A real-time audio spectrum analyzer and function generator demo for the BSc Analog Electronics open day. Prospective students interact with the demo by speaking into a microphone, or by observing waveforms generated by an onboard function generator — all visualized on a big screen.

The system uses an **Olimex ESP32-ADF** board as the audio hardware frontend (microphone + function generator via ES8388 codec) connected physically to a **Raspberry Pi 5** via USB serial. The Pi processes audio, runs a web server, and streams visuals over WiFi to any browser on the local network (big TV, laptop, tablet, phone).

## Hardware Details

### Olimex ESP32-ADF Board

- **MCU**: ESP32-WROVER-B (dual-core 240MHz, 8MB PSRAM, 4MB Flash)
- **Audio Codec**: ES8388 — 24-bit stereo codec with both ADC and DAC
  - Connected to ESP32 via **I2S** (audio data) and **I2C** (control/configuration)
  - DAC output routed to 3.5mm jack and 2x3W speaker amplifier
  - ADC input routed from onboard stereo electret microphones
  - Supports full-duplex (simultaneous record + playback)
- **USB**: Micro-USB with USB-UART bridge (for programming and serial data)
- **Other**: 4 touch buttons, 3 tactile buttons, IR receiver, UEXT connector, LiPo charger
- **Olimex GitHub**: https://github.com/OLIMEX/ESP32-ADF

### Raspberry Pi 5

- Runs Raspberry Pi OS (64-bit)
- Connected to ESP32 via USB (appears as /dev/ttyUSB0 or /dev/ttyACM0)
- Connected to local WiFi network
- Python 3.11+ environment

### Display

- Any browser-capable screen on the same WiFi network
- Recommended: large TV (32"+) via HDMI or smart TV browser
- Visitors can also view on their phones via QR code

## Software Components

### 1. ESP32 Firmware (C — ESP-IDF)

**Location**: `esp32-firmware/`

**Responsibilities**:
- Initialize and configure ES8388 codec via I2C
- Capture audio from microphones via I2S (16kHz, 16-bit, mono)
- Generate waveforms via I2S DAC output (sine, square, triangle, sawtooth)
- Receive control commands from Pi over USB serial
- Send raw audio samples to Pi over USB serial
- Handle multiplexing of commands and audio data on the serial link

**Key ESP-IDF components**:
- `driver/i2s` — I2S for audio in/out with ES8388
- `driver/i2c` — I2C for ES8388 register configuration
- `driver/uart` — USB serial communication with Pi
- `esp_dsp` — (optional) DSP functions if needed on ESP32 side

**Waveform generation (DDS — Direct Digital Synthesis)**:
- Maintain a phase accumulator
- Increment phase each sample by `(frequency / sample_rate) * 2 * PI`
- Look up or compute the waveform value for current phase
- Write samples to I2S DAC buffer
- Supported waveforms: sine, square, triangle, sawtooth
- Parameters: waveform type, frequency (20Hz-20kHz), amplitude (0.0-1.0)

**ES8388 configuration**:
- Set sample rate (16kHz for mic, configurable for DAC)
- Configure ADC input to microphone pins (LIN1/RIN1 or LIN2/RIN2)
- Configure DAC output to 3.5mm jack and/or speakers
- Set gain levels for mic preamp and DAC output
- Enable full-duplex mode

**File structure**:
```
esp32-firmware/
├── CMakeLists.txt
├── sdkconfig.defaults          # ESP-IDF config (PSRAM, flash size, etc.)
├── main/
│   ├── CMakeLists.txt
│   ├── main.c                  # App entry point, task creation
│   ├── es8388.c                # ES8388 driver (I2C register config)
│   ├── es8388.h
│   ├── audio_io.c              # I2S setup, mic capture, DAC output
│   ├── audio_io.h
│   ├── funcgen.c               # DDS waveform generator
│   ├── funcgen.h
│   ├── serial_comm.c           # UART protocol handler
│   └── serial_comm.h
```

**FreeRTOS tasks**:
- **Task 1 (Core 0)**: Waveform generation --> I2S DAC write
- **Task 2 (Core 1)**: I2S ADC read (mic) --> serial transmit to Pi
- **Task 3**: Serial command receiver (parse commands from Pi)

### 2. Serial Protocol

The USB serial link carries both control commands (Pi to ESP32) and audio data (ESP32 to Pi) over the same physical connection.

**Baud rate**: 921600 (required for 16kHz 16-bit audio = 256kbps raw)

#### Commands (Pi to ESP32) — Text-based, newline-terminated

```
WAVE <type> <frequency_hz> <amplitude>
  type: SINE | SQUARE | TRIANGLE | SAW
  frequency_hz: 20-20000 (integer)
  amplitude: 0.0-1.0 (float)
  Example: WAVE SINE 1000 0.8

SWEEP <start_hz> <end_hz> <duration_sec>
  Example: SWEEP 100 10000 5

STOP
  Stop function generator output (silence)

STATUS
  Request current state (ESP32 responds with status line)
```

#### Audio Data (ESP32 to Pi) — Binary packets

```
+----------+------------+----------------------+----------+
| SYNC (2B)| LENGTH (2B)| PCM SAMPLES (N bytes)| CRC8 (1B)|
+----------+------------+----------------------+----------+

SYNC:    0xAA 0x55 (magic bytes to identify start of audio packet)
LENGTH:  uint16 little-endian, number of PCM bytes following
SAMPLES: int16 little-endian PCM samples (16-bit signed, mono, 16kHz)
CRC8:    simple CRC over the PCM bytes for error detection
```

**Packet size**: 512 samples per packet (1024 bytes PCM) at ~32 packets/sec at 16kHz. This balances latency (~32ms per packet) with serial efficiency.

#### Status Response (ESP32 to Pi) — Text-based

```
STATUS:<wave_type>,<frequency>,<amplitude>,<running>
  Example: STATUS:SINE,1000,0.8,1
```

Text lines and binary packets are distinguished by the sync bytes — the parser on the Pi side looks for 0xAA 0x55 for audio and treats anything else as text.

### 3. Pi Server (Python)

**Location**: `pi-server/`

**Responsibilities**:
- Read serial port: parse audio packets and status messages from ESP32
- Compute FFT on incoming audio (numpy)
- Serve web interface (HTML/JS/CSS)
- Push real-time data to browser via WebSocket (waveform + FFT bins)
- Forward user control commands to ESP32 via serial
- Optionally: read from future ADC input

**Key Python packages**:
- `pyserial` — serial communication with ESP32
- `numpy` — FFT computation
- `asyncio` + `websockets` — async WebSocket server
  OR `FastAPI` + `uvicorn` with WebSocket support
- Standard library for HTTP static file serving

**File structure**:
```
pi-server/
├── requirements.txt
├── server.py               # Main entry point
├── serial_handler.py       # Serial port reader/writer, packet parser
├── audio_processor.py      # FFT, windowing, spectrogram buffer
├── websocket_server.py     # WebSocket endpoint for browser
├── static/
│   ├── index.html          # Main visualizer page
│   ├── visualizer.js       # Canvas/WebGL rendering
│   └── style.css           # Styling
```

**Data flow**:
```
Serial port --> packet parser --> raw PCM buffer
                                      |
                            +---------+---------+
                            v                   v
                      Waveform buffer      FFT (numpy)
                      (last N samples)     (windowed, magnitude)
                            |                   |
                            +---------+---------+
                                      v
                            WebSocket broadcast
                            (JSON: waveform + fft bins)
                                      |
                                      v
                                Browser(s)
```

**WebSocket message format (Pi to Browser)**:

```json
{
  "type": "audio",
  "waveform": [0.12, -0.34, "..."],
  "fft_magnitude": [0.01, 0.5, "..."],
  "fft_frequencies": [0, 31.25, "..."],
  "sample_rate": 16000
}
```

**WebSocket message format (Browser to Pi to ESP32)**:

```json
{
  "type": "command",
  "command": "WAVE SINE 1000 0.8"
}
```

**Update rate**: Target 30 fps for smooth visuals. Each frame = one FFT window.

## Key Technical Considerations

### ES8388 I2C Register Configuration
The ES8388 requires careful register setup. Key registers:
- Power management (CHIPPOWER, ADCPOWER, DACPOWER)
- ADC/DAC format and sample rate (MASTERMODE, ADCCONTROL, DACCONTROL)
- Input selection (MIC vs LINE-IN routing)
- Volume/gain control
- Mixer configuration

Reference: ES8388 datasheet + Olimex ESP32-ADF source code on GitHub.

### Audio Latency Budget
Target: under 100ms end-to-end (generation/capture to screen)
- ESP32 I2S buffer fill: ~16ms (512 samples at 16kHz)
- Serial transmission: ~11ms (1024 bytes at 921600 baud)
- Pi FFT computation: under 1ms
- WebSocket transmission: under 5ms (local WiFi)
- Browser render: ~16ms (at 60fps)
- **Total: ~50ms** — well within target

### Error Handling
- Serial CRC errors: drop packet, wait for next sync
- Serial disconnection: Pi auto-reconnects, browser shows "reconnecting" overlay
- WebSocket disconnection: browser auto-reconnects with exponential backoff
- ESP32 watchdog: restart tasks if audio pipeline stalls

## Analog Electronics Talking Points

This demo showcases the following concepts from the curriculum:

1. **MEMS microphone**: capacitive transducer, charge amplifier
2. **Anti-aliasing filter**: low-pass filtering before ADC sampling
3. **ADC / DAC**: sample-and-hold, quantization, reconstruction
4. **Sampling theorem**: Nyquist frequency, aliasing (demonstrable live)
5. **Fourier analysis**: frequency decomposition of real signals
6. **DDS waveform generation**: digital-to-analog signal synthesis
7. **Signal conditioning**: gain, attenuation, DC biasing
8. **Noise**: thermal noise visible on the spectrum floor
9. **Nonlinear distortion**: clipping leads to harmonics visible on FFT
10. **Filtering** (with breadboard): RC/RLC circuits attenuating frequencies

### Prepared Demo Scenarios
- **Voice**: speak or clap, see waveform + spectrum react
- **Pure tone**: function generator sine wave produces single FFT peak
- **Harmonics**: switch to square wave and odd harmonics appear
- **Frequency sweep**: animated peak moving across spectrum
- **Clipping demo** (future, with ADC): overdrive a signal, watch harmonics explode
- **Filter demo** (future, with ADC): RC low-pass on breadboard, see high frequencies vanish
